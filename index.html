1. PENGERTIAN POLICY ROUTING

Dari sumber MikroTik Documentation (2024), policy routing adalah metode untuk mengarahkan lalu lintas yang memenuhi kriteria tertentu ke gateway tertentu. Metode ini dapat digunakan untuk memaksa beberapa pelanggan atau protokol spesifik dari server (misalnya lalu lintas HTTP) agar selalu dirutekan ke gateway tertentu. Policy routing bahkan dapat digunakan untuk mengarahkan lalu lintas lokal dan luar negeri ke gateway yang berbeda.

Dari sumber Network Troubleshooting Best Practices (2023), ketika keputusan perutean menggunakan informasi tambahan, seperti alamat sumber paket, hal ini disebut policy routing. Policy routing diimplementasikan sebagai daftar aturan perutean kebijakan, yang memilih tabel perutean berbeda berdasarkan alamat tujuan, alamat sumber, antarmuka sumber, dan penanda perutean (yang dapat diubah oleh aturan mangle firewall) dari paket.

Manfaat Policy Routing

  • Pemisahan Lalu Lintas: Mengarahkan lalu lintas ke gateway berbeda berdasarkan jenis aplikasi atau alamat sumber.
  • Optimalisasi Biaya: Mengarahkan lalu lintas dengan kebutuhan bandwidth tinggi melalui koneksi yang lebih ekonomis.
  • Peningkatan Performa: Mengarahkan lalu lintas sensitif latensi melalui jalur dengan kualitas koneksi terbaik.
  • Redundansi Koneksi: Memungkinkan penggunaan beberapa koneksi internet untuk kebutuhan spesifik.
Ilustrasi konsep Policy Routing di MikroTik Router

Dari sumber NetworkTik (2024), penting untuk memahami bahwa dalam MikroTik terdapat 2 tabel ketika berbicara tentang perutean: RIB (Routing Information Base) dan FIB (Forwarding Information Base). RIB terutama berisi semua rute dari protokol perutean yang Anda implementasikan. Jika Anda tidak menggunakan penanda perutean apa pun, maka semua rute pada router MikroTik akan berada di dalam tabel perutean utama. Jika Anda menggunakan beberapa penanda perutean, maka RIB akan berisi rute yang dikelompokkan dalam tabel perutean terpisah berdasarkan nilai penanda peruteannya. Dari tabel RIB, rute terbaik dipilih dan akan diinstal dalam tabel FIB di mana keputusan perutean akan dibuat. Itu berarti bahwa tidak ada keputusan rute yang dibuat pada tabel RIB.

Dari 2 pendapat tersebut dapat disimpulkan bahwa policy routing adalah metode perutean lanjutan di MikroTik yang memungkinkan pengguna untuk mengarahkan lalu lintas jaringan berdasarkan berbagai kriteria seperti alamat sumber, alamat tujuan, antarmuka, atau penanda khusus, bukan hanya berdasarkan alamat tujuan seperti pada perutean tradisional. Hal ini memberikan fleksibilitas dan kontrol yang lebih besar terhadap jalur yang diambil oleh paket data dalam jaringan.

2. MEMBUAT TABEL ROUTING TAMBAHAN

# topologi yang digunakan

Tampilan topologi di gns3

Dari sumber MikroTik Documentation (2024), tabel routing dapat dilihat dan dikonfigurasi dari menu /routing/table. Secara default, RouterOS hanya memiliki tabel routing 'main'. Jika tabel routing kustom diperlukan, tabel tersebut harus didefinisikan terlebih dahulu di menu ini sebelum digunakan di bagian mana pun dalam konfigurasi.

Dari sumber MikroTik Wiki (2023), semua rute secara default disimpan dalam tabel routing utama. Rute dapat ditetapkan ke tabel routing tertentu dengan mengatur properti routing-mark mereka ke nama tabel routing lain. Tabel routing direferensikan berdasarkan namanya, dan dibuat secara otomatis ketika direferensikan dalam konfigurasi. Setiap tabel routing hanya dapat memiliki satu rute aktif untuk setiap nilai prefiks IP dst-address.

Langkah-langkah Membuat Tabel Routing Tambahan Menggunakan Winbox

Berikut adalah langkah-langkah untuk membuat tabel routing tambahan dengan menggunakan Winbox:

  1. Buka aplikasi Winbox dan hubungkan ke router MikroTik Anda.
  2. Pada menu sebelah kiri, navigasikan ke Routing.
  3. Klik pada tab Tables.
  4. Klik tombol + (plus) untuk menambahkan tabel routing baru.
  5. Pada jendela "New Route Table", masukkan nama tabel routing yang diinginkan, misalnya "ISP1" atau "ISP2".
  6. Centang opsi Fib (Forwarding Information Base).
  7. Klik tombol OK untuk menyimpan konfigurasi.
Membuat tabel routing baru menggunakan Winbox
# Hasil dari pembuatan tabel routing
/routing table print
Flags: * - default, D - dynamic, X - disabled, I - invalid, U - used
# NAME FIB
0 * main yes
1 ISP1 yes
2 ISP2 yes

Pastikan Anda mengaktifkan opsi FIB (Forwarding Information Base) ketika membuat tabel routing baru. Opsi ini memungkinkan tabel routing tersebut digunakan untuk pengambilan keputusan perutean paket data.

Dari 2 pendapat tersebut dapat disimpulkan bahwa tabel routing adalah komponen fundamental dalam implementasi policy routing. MikroTik RouterOS secara default hanya memiliki tabel routing 'main', tetapi memungkinkan pembuatan tabel routing tambahan untuk kebutuhan policy routing. Setiap tabel routing dapat berisi serangkaian rute sendiri yang mengarahkan tujuan yang sama ke gateway yang berbeda, memberikan fleksibilitas dalam perutean lalu lintas jaringan.

3. MENAMBAHKAN RUTE PADA TABEL ROUTING

Dari sumber MikroTik Documentation (2024), setelah membuat tabel routing kustom, langkah selanjutnya adalah menambahkan rute ke dalamnya. Misalnya, jika ingin mengarahkan lalu lintas ke 8.8.8.8 melalui gateway 172.16.2.1 di tabel routing 'myTable', gunakan perintah: /ip route add dst-address=8.8.8.8 gateway=172.16.2.1 routing-table=myTable. Namun, konfigurasi ini saja tidak cukup, kita memerlukan metode untuk memaksa lalu lintas agar benar-benar menggunakan tabel yang baru dibuat.

Dari sumber Tech Layer-X (2025), untuk menambahkan rute statis menggunakan Winbox, buka Winbox dan masuk ke router MikroTik Anda. Navigasikan ke IP > Routes. Klik tombol Add (+) untuk membuka jendela "New Route". Masukkan parameter yang diperlukan dan klik OK untuk menyimpan rute.

Langkah-langkah Menambahkan Rute pada Tabel Routing Menggunakan Winbox

Berikut adalah langkah-langkah untuk menambahkan rute ke tabel routing yang telah dibuat:

  1. Dalam aplikasi Winbox, navigasikan ke IP > Routes.
  2. Klik tombol + (plus) untuk menambahkan rute baru.
  3. Pada jendela "New Route" yang muncul, konfigurasi parameter berikut:
    • Dst. Address: Alamat tujuan (untuk rute default, gunakan 0.0.0.0/0)
    • Gateway: Alamat IP gateway ISP pertama (misalnya 192.168.1.1)
    • Routing Mark: Biarkan kosong (ini adalah rute di tabel utama)
    • Distance: 1 (prioritas rute)
    • Check Gateway: ping (opsional, untuk memantau ketersediaan gateway)
  4. Klik OK untuk menyimpan rute default di tabel main.
  5. Ulangi langkah 2-4 untuk menambahkan rute default ke tabel ISP1:
    • Dst. Address: 0.0.0.0/0
    • Gateway: Alamat IP gateway ISP pertama
    • Routing Table: Pilih "ISP1" dari dropdown
    • Distance: 1
  6. Ulangi lagi untuk tabel ISP2 dengan gateway ISP kedua.
Menambahkan rute default ke tabel routing kustom di MikroTik
# Hasil dari penambahan rute
/ip route print
Flags: D - dynamic, X - disabled, A - active, C - connect, S - static, r - rip, b - bgp, o - ospf, m - mme,
B - blackhole, U - unreachable, P - prohibit
# DST-ADDRESS PREF-SRC GATEWAY DISTANCE
0 ADS 0.0.0.0/0 192.168.1.1 1
1 ADS 0.0.0.0/0 192.168.2.1 1 routing-table=ISP1
2 ADS 0.0.0.0/0 192.168.3.1 1 routing-table=ISP2

Pastikan bahwa Anda memiliki koneksi fisik yang benar dan alamat IP gateway yang akurat untuk setiap ISP. Penggunaan opsi Check Gateway sangat direkomendasikan untuk memastikan rute hanya aktif jika gateway dapat dijangkau.

Dari 2 pendapat tersebut dapat disimpulkan bahwa setelah membuat tabel routing kustom, langkah berikutnya adalah menambahkan rute ke dalamnya menggunakan Winbox. Meskipun kita telah membuat tabel routing dan menambahkan rute ke dalamnya, konfigurasi ini saja belum cukup untuk mengimplementasikan policy routing. Kita masih memerlukan langkah tambahan untuk menentukan kriteria lalu lintas mana yang akan menggunakan tabel routing tertentu.

4. MEMBUAT ATURAN MANGLE UNTUK PENANDAAN PAKET

# topologi yang digunakan

Tampilan topologi di gns3

Dari sumber MikroTik Documentation (2024), RouterOS memberikan opsi untuk menggunakan firewall mangle untuk policy routing. Metode ini memberikan kontrol lebih atas kriteria yang digunakan untuk mengarahkan lalu lintas, misalnya, penyeimbangan per koneksi atau per paket, dan sebagainya. Untuk informasi lebih lanjut tentang cara menggunakan penandaan mangle, lihat contoh Firewall Marking.

Dari sumber NetworkTik (2024), bayangkan Anda memiliki 2 koneksi ISP yang terhubung ke router MikroTik Anda, dan Anda memutuskan bahwa semua lalu lintas game yang berasal dari LAN harus melalui ISP2 dan semua lalu lintas non-game lainnya harus melalui ISP1.

Langkah-langkah Membuat Aturan Mangle Menggunakan Winbox

Berikut adalah langkah-langkah untuk membuat aturan mangle yang akan menandai paket berdasarkan alamat IP sumber:

  1. Dalam aplikasi Winbox, navigasikan ke IP > Firewall.
  2. Pilih tab Mangle.
  3. Klik tombol + (plus) untuk menambahkan aturan mangle baru.
  4. Pada jendela "New Mangle Rule" yang muncul, konfigurasi tab General:
    • Chain: prerouting (karena kita ingin menandai paket sebelum keputusan routing)
    • Src. Address: Masukkan rentang alamat IP klien yang ingin Anda arahkan ke ISP1 (misalnya 192.168.0.1-192.168.0.100)
  5. Klik pada tab Action dan konfigurasi:
    • Action: mark-routing
    • New Routing Mark: ISP1 (sesuai dengan nama tabel routing)
    • Passthrough: Yes (memungkinkan paket untuk melanjutkan melalui aturan mangle lain)
  6. Klik OK untuk menyimpan aturan mangle pertama.
  7. Ulangi langkah 3-6 untuk membuat aturan kedua untuk ISP2, dengan mengubah Src. Address menjadi rentang IP klien kedua (misalnya 192.168.0.101-192.168.0.200) dan New Routing Mark menjadi "ISP2".

Anda juga dapat menandai paket berdasarkan kriteria lain seperti protokol, port, atau aplikasi tertentu. Misalnya, untuk mengarahkan lalu lintas video streaming melalui ISP1 dan lalu lintas game melalui ISP2.

# Hasil dari pembuatan aturan mangle
/ip firewall mangle print
Flags: X - disabled, I - invalid, D - dynamic
0 chain=prerouting action=mark-routing new-routing-mark=ISP1 passthrough=yes
src-address=192.168.0.1-192.168.0.100 log=no log-prefix=""
1 chain=prerouting action=mark-routing new-routing-mark=ISP2 passthrough=yes
src-address=192.168.0.101-192.168.0.200 log=no log-prefix=""

Dari 2 pendapat tersebut dapat disimpulkan bahwa aturan mangle adalah komponen kunci dalam implementasi policy routing di MikroTik. Dengan menggunakan aturan mangle, kita dapat menandai paket berdasarkan berbagai kriteria seperti alamat IP sumber, jenis lalu lintas, atau aplikasi, kemudian mengarahkannya ke tabel routing yang sesuai. Aturan mangle memberikan fleksibilitas tinggi dalam mendefinisikan kriteria untuk policy routing.

5. MEMBUAT ATURAN ROUTING RULES

# topologi yang digunakan

Tampilan topologi di gns3

Dari sumber MikroTik Documentation (2024), selain firewall mangle, RouterOS juga menyediakan routing rules - serangkaian parameter dasar yang dapat digunakan untuk dengan cepat mengarahkan lalu lintas. Tidak direkomendasikan untuk menggunakan kedua metode (mangle dan routing rules) secara bersamaan kecuali Anda tahu persis apa yang Anda lakukan. Jika Anda benar-benar perlu menggunakan mangle dan routing rules dalam setup yang sama, perhatikan bahwa mangle memiliki prioritas lebih tinggi, artinya jika lalu lintas yang ditandai mangle dapat diselesaikan dalam tabel, maka aturan rute tidak akan pernah melihat lalu lintas ini.

Dari sumber Network Troubleshooting Best Practices (2023), policy routing diimplementasikan sebagai daftar aturan perutean kebijakan, yang memilih tabel perutean berbeda berdasarkan alamat tujuan, alamat sumber, antarmuka sumber, dan penanda perutean (yang dapat diubah oleh aturan mangle firewall) dari paket. FIB menggunakan informasi berikut dari paket untuk menentukan tujuannya: memeriksa bahwa paket harus dikirimkan secara lokal (alamat tujuan adalah alamat router). Aturan yang tidak cocok dengan paket saat ini diabaikan. Jika suatu aturan memiliki aksi drop atau unreachable, maka itu dikembalikan sebagai hasil dari proses keputusan perutean. Jika aksinya lookup, maka alamat tujuan paket dicari dalam tabel routing yang ditentukan dalam aturan. Jika pencarian gagal (tidak ada rute yang cocok dengan alamat tujuan paket), maka FIB melanjutkan ke aturan berikutnya.

Langkah-langkah Membuat Aturan Routing Rules Menggunakan Winbox

Setelah membuat aturan mangle untuk menandai paket, langkah berikutnya adalah membuat routing rules yang akan mengarahkan paket bertanda ke tabel routing yang sesuai:

  1. Dalam aplikasi Winbox, navigasikan ke IP > Routes.
  2. Pilih tab Rules.
  3. Klik tombol + (plus) untuk menambahkan aturan routing baru.
  4. Pada jendela "New Route Rule" yang muncul, konfigurasi:
    • Routing Mark: ISP1 (sesuai dengan mark yang ditetapkan dalam aturan mangle)
    • Action: lookup
    • Table: ISP1
  5. Klik OK untuk menyimpan aturan routing pertama.
  6. Ulangi langkah 3-5 untuk membuat aturan kedua untuk ISP2, dengan Routing Mark dan Table diatur ke "ISP2".
  7. Tambahkan aturan ketiga untuk lalu lintas yang tidak tertandai:
    • Routing Mark: biarkan kosong
    • Action: lookup
    • Table: main
Membuat aturan routing di MikroTik

Urutkan aturan routing dengan benar sangatlah penting. Aturan diproses dari atas ke bawah, dan pemrosesan berhenti saat aturan pertama yang cocok ditemukan. Pastikan untuk menempatkan aturan yang lebih spesifik di atas aturan yang lebih umum.

# Hasil dari pembuatan aturan routing
/ip route rule print
Flags: X - disabled, I - invalid
0 routing-mark=ISP1 action=lookup table=ISP1
1 routing-mark=ISP2 action=lookup table=ISP2
2 action=lookup table=main

Dari 2 pendapat tersebut dapat disimpulkan bahwa routing rules adalah komponen penting dalam implementasi policy routing yang bekerja sama dengan aturan mangle. Routing rules menentukan tabel routing mana yang akan digunakan untuk paket yang telah ditandai oleh aturan mangle. Penting untuk memahami urutan prioritas antara mangle dan routing rules, serta bagaimana FIB memproses aturan untuk menentukan jalur pengiriman paket.

6. PENGUJIAN DAN VERIFIKASI POLICY ROUTING

# topologi yang digunakan

Tampilan topologi di gns3

Dari sumber MikroTik Wiki (2023), Winbox dapat digunakan sebagai alat untuk memantau lalu lintas setiap antarmuka, queue, atau aturan firewall secara real-time. Ini sangat berguna untuk memverifikasi bahwa policy routing bekerja sebagaimana mestinya.

Dari sumber Tech Layer-X (2025), ketika menggunakan aturan mangle untuk policy routing, pastikan bahwa aturan fasttrack tidak secara tidak sengaja melewati paket-paket yang telah ditandai. Policy routing dengan tabel routing terpisah dan penggunaan aturan mangle adalah teknik-teknik lanjutan yang penting dalam konfigurasi jaringan MikroTik.

Langkah-langkah Menguji Policy Routing Menggunakan Winbox

Berikut adalah langkah-langkah untuk memverifikasi bahwa policy routing bekerja dengan benar:

  1. Periksa tabel routing dengan navigasi ke IP > Routes dan pastikan semua rute yang Anda konfigurasi memiliki status "A" (active).
  2. Verifikasi aturan mangle dengan navigasi ke IP > Firewall > tab Mangle dan periksa kolom "Bytes" dan "Packets" untuk melihat apakah aturan Anda menangkap lalu lintas.
  3. Gunakan Torch untuk memantau lalu lintas secara real-time:
    • Navigasikan ke Tools > Torch
    • Pilih antarmuka yang ingin Anda pantau (misalnya, interface ke ISP1)
    • Klik Start untuk memulai pemantauan
    • Lakukan hal yang sama untuk antarmuka ke ISP2
  4. Uji konektivitas dari komputer klien yang berada dalam rentang IP yang berbeda:
    • Dari komputer dengan alamat IP dalam rentang ISP1, buka Command Prompt dan jalankan tracert 8.8.8.8
    • Lakukan hal yang sama dari komputer dengan alamat IP dalam rentang ISP2
    • Bandingkan hasilnya - traceroute harus menunjukkan gateway berbeda untuk masing-masing komputer
Memantau lalu lintas menggunakan Torch di MikroTik

Anda juga dapat menggunakan fitur "Connection Tracking" untuk melihat koneksi aktif dan jalur yang digunakan. Navigasikan ke IP > Firewall > tab Connections untuk melihat daftar koneksi saat ini.

1 1 ms 1 ms 1 ms 192.168.0.1
2 5 ms 5 ms 6 ms 192.168.1.1
3 15 ms 14 ms 15 ms ... (ISP1 path)
# Hasil traceroute dari klien di rentang IP ISP2
Tracing route to 8.8.8.8 over a maximum of 30 hops:
1 1 ms 1 ms 1 ms 192.168.0.1
2 7 ms 7 ms 8 ms 192.168.3.1
3 20 ms 18 ms 19 ms ... (ISP2 path)

Dari 2 pendapat tersebut dapat disimpulkan bahwa pengujian dan verifikasi adalah langkah penting setelah mengonfigurasi policy routing untuk memastikan bahwa lalu lintas diarahkan melalui jalur yang diinginkan. MikroTik menyediakan berbagai alat seperti Torch, Connection Tracking, dan statistik aturan mangle untuk membantu administrator jaringan memverifikasi bahwa policy routing bekerja dengan benar. Perlu diperhatikan juga interaksi antara policy routing dan fitur lain seperti FastTrack yang dapat mempengaruhi perilaku routing.

7. TROUBLESHOOTING POLICY ROUTING

# topologi yang digunakan

Tampilan topologi di gns3

Dari sumber MikroTik Documentation (2024), jika terlalu banyak perubahan dibuat saat dalam safe mode, dan tidak ada ruang dalam riwayat untuk menampung semuanya (saat ini riwayat menyimpan hingga 100 tindakan terbaru), maka sesi secara otomatis dikeluarkan dari safe mode, dan tidak ada perubahan yang secara otomatis dibatalkan. Oleh karena itu, lebih baik mengubah konfigurasi dalam langkah-langkah kecil, saat dalam safe mode. Menekan [Ctrl]+[X] dua kali adalah cara mudah untuk mengosongkan daftar tindakan safe mode.

Dari sumber Tech Layer-X (2025), ketika menggunakan aturan mangle untuk policy routing, pastikan bahwa aturan fasttrack tidak secara tidak sengaja melewati paket-paket yang telah ditandai. Penting untuk memahami teknik-teknik lanjutan seperti redistribusi rute, policy routing dengan tabel routing terpisah, dan penggunaan aturan mangle. Praktik terbaik untuk merancang jaringan yang kuat dan aman termasuk menggunakan antarmuka loopback, memprioritaskan rute untuk ketersediaan tinggi, dan memantau kinerja perutean dengan benar. Serta memberikan tips troubleshooting untuk membantu mendiagnosis dan menyelesaikan masalah perutean umum di jaringan MikroTik.

Masalah Umum dan Solusinya

Berikut adalah beberapa masalah umum yang mungkin Anda hadapi saat mengimplementasikan policy routing dan solusinya:

Masalah 1: Lalu lintas tidak mengikuti aturan policy routing
# Penyebab potensial dan solusi:
1. Aturan mangle tidak cocok dengan lalu lintas - periksa konfigurasi src-address, dst-address, dll.
2. Urutan aturan routing rules tidak benar - pastikan aturan khusus berada di atas aturan umum.
3. FastTrack mengabaikan penandaan - nonaktifkan FastTrack atau pastikan mengecualikan lalu lintas yang perlu ditandai.
Masalah 2: Salah satu gateway tidak tersedia
# Penyebab potensial dan solusi:
1. Tambahkan fitur check-gateway pada rute untuk mendeteksi kegagalan gateway.
2. Implementasikan failover dengan menambahkan rute cadangan dengan distance yang lebih tinggi.
Masalah 3: Aplikasi tertentu tidak berfungsi dengan benar
# Penyebab potensial dan solusi:
1. Beberapa aplikasi memerlukan koneksi balik (return path) yang sesuai - gunakan connection-mark untuk menandai paket return.
2. Masalah NAT - pastikan konfigurasi NAT kompatibel dengan policy routing Anda.

Langkah-langkah Troubleshooting Menggunakan Winbox

  1. Gunakan Safe Mode:
    • Klik tombol "Safe Mode" di Winbox sebelum membuat perubahan konfigurasi.
    • Jika perubahan menyebabkan masalah, Anda dapat dengan mudah membatalkannya.
  2. Periksa Log Router:
    • Navigasikan ke Log di menu sebelah kiri Winbox.
    • Cari pesan error atau peringatan terkait routing atau firewall.
  3. Verifikasi Connection Tracking:
    • Navigasikan ke IP > Firewall > tab Connections.
    • Periksa apakah koneksi ditandai dengan benar dan menggunakan routing mark yang diharapkan.
  4. Nonaktifkan Sementara Fitur Yang Berpotensi Mengganggu:
    • Nonaktifkan FastTrack: IP > Firewall > Filter Rules > nonaktifkan aturan FastTrack.
    • Sederhanakan konfigurasi NAT untuk pengujian.
Menggunakan Safe Mode di Winbox untuk troubleshooting yang aman

Selalu buat backup konfigurasi sebelum melakukan troubleshooting ekstensif. Navigasikan ke Files > Backup di Winbox dan buat file backup yang dapat Anda pulihkan jika terjadi masalah serius.

Dari 2 pendapat tersebut dapat disimpulkan bahwa troubleshooting policy routing membutuhkan pendekatan sistematis dan pemahaman mendalam tentang interaksi antara berbagai komponen MikroTik seperti firewall, NAT, dan FastTrack. Menggunakan Safe Mode saat melakukan perubahan konfigurasi, memantau log router, dan memahami alat diagnostik yang disediakan MikroTik adalah kunci untuk mendiagnosis dan menyelesaikan masalah. Penting juga untuk memahami bagaimana fitur-fitur lain seperti FastTrack dapat mempengaruhi fungsionalitas policy routing.
